**src/core_api/agents/freeform_crew.py:**
- Import: `from crewai import Agent, Crew, Process, Task, LLM`
- Import: `from crewai.project import CrewBase, agent, crew, task`

- Import: `from logging_config import get_logger`

- `@CrewBase` class `FreeformCrew`:
  - `agents_config = "config/agents.yaml"`
  - `tasks_config = "config/tasks.yaml"`
  - `def _llm(self) -> LLM:` returns `LLM(model="openai/reasoning-model", base_url=os.getenv("LITELLM_BASE_URL", "http://litellm:4000/v1"), api_key=os.getenv("LITELLM_MASTER_KEY", "sk-maai-local"), stream=False, timeout=60.0)` — per D-07 (reasoning-model alias), D-08 (stream=False for AGNT-08)
  - `@agent def freeform_agent(self) -> Agent:` returns `Agent(config=self.agents_config["freeform_agent"], llm=self._llm(), max_iter=5, max_execution_time=60, memory=False, verbose=False)` — per D-09 (AGNT-09 guardrails)
  - `@task def freeform_task(self) -> Task:` returns `Task(config=self.tasks_config["freeform_task"])`
  - `@crew def crew(self) -> Crew:` returns `Crew(agents=self.agents, tasks=self.tasks, process=Process.sequential, embedder={"provider": "ollama", "config": {"model_name": os.getenv("EMBEDDING_MODEL", "nomic-embed-text"), "url": "http://ollama:11434/api/embeddings"}})` — per D-10 (AGNT-07 explicit Ollama embedder; uses Ollama directly because CrewAI's ollama embedder provider does not support LiteLLM proxy for embeddings)

- Module-level function `def run_freeform_crew(messages: list, user_message: str) -> str:`:
  1. Log "Running freeform crew" with message count
  2. Create `FreeformCrew().crew()`
  3. Format history: `"\n".join(f"{m.role}: {m.content}" for m in messages[:-1])` — exclude latest user msg (it's duplicated in user_message)
  4. `result = crew.kickoff(inputs={"messages": history or "(no prior messages)", "user_message": user_message})`
  5. Log completion
  6. `return str(result)`

Create src/core_api/Dockerfile:

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# Install uv for fast dependency installation
RUN pip install --no-cache-dir uv

# Copy project metadata and install dependencies
COPY pyproject.toml .
RUN uv pip install --system --no-cache .

# Copy application code
COPY . .

# Create uploads directory (mounted as volume in compose)
RUN mkdir -p /app/uploads

EXPOSE 8000

# Health check uses Python stdlib (no curl/wget needed)
HEALTHCHECK --interval=15s --timeout=5s --retries=3 --start-period=30s \
  CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')"

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

Key decisions:
- python:3.11-slim base image (per CLAUDE.md: Python 3.11 only)
- uv for fast dependency installation (per CLAUDE.md: use uv, not pip)
- Health check uses Python urllib (no extra binary needed)
- Exposes port 8000
- CMD uses uvicorn pointing to main:app
