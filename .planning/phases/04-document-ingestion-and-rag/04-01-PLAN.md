**src/docproc/main.py** ‚Äî FastAPI application with:

1. Pydantic models:
   - `ProcessRequest(BaseModel)`: `file_path: str`, `ocr_enabled: bool = True`
   - `PageResult(BaseModel)`: `page_no: int`, `text: str`, `has_ocr: bool`
   - `ProcessResponse(BaseModel)`: `status: str`, `file_name: str`, `pages: list[PageResult]`, `full_text: str`, `total_pages: int`

2. Module-level converter setup in lifespan:
   - Create TWO `DocumentConverter` instances cached in `app.state`:
     a. `converter_ocr` ‚Äî with `PdfPipelineOptions(do_ocr=True, do_table_structure=True, ocr_options=EasyOcrOptions(lang=["en"], use_gpu=False, force_full_page_ocr=True))` wrapped in `PdfFormatOption` for `InputFormat.PDF`
     b. `converter_text` ‚Äî plain `DocumentConverter()` with no OCR (faster for text PDFs)
   - In lifespan startup, pre-warm EasyOCR by instantiating `easyocr.Reader(["en"], gpu=False)` to trigger model download (Research Pitfall 4). Log "EasyOCR models pre-loaded" on success.
   - `use_gpu` is read from env var `DOCPROC_USE_GPU` (default "false"), parsed as bool.

NOTE on DOCP-02: Using EasyOCR (not PaddleOCR) per RESEARCH.md finding that PaddleOCR is not a native Docling backend. EasyOCR is the practical equivalent within Docling's OCR framework.

3. `POST /process` endpoint:
   - Validate `file_path` exists, return 404 if not
   - Validate file extension is `.pdf`, `.png`, `.jpg`, `.jpeg`, `.tiff`, `.bmp` ‚Äî return 400 for unsupported
   - Choose `converter_ocr` if `ocr_enabled=True`, else `converter_text`
   - Call `converter.convert(file_path)`, get `result.document`
   - Iterate `doc.iterate_items()`: for each element with `.text` attribute, extract text and page number from `element.prov[0].page_no` (default 0 if no prov). Per D-04, this handles both text and scanned PDFs.
   - Build `PageResult` list grouped by page number
   - Build `full_text` by joining all page texts with newlines
   - Return `ProcessResponse` with status="success"
   - Wrap in try/except: on failure return `{"status": "error", "detail": str(exc)}` with HTTP 500

4. `GET /health` endpoint returning `{"status": "ok"}`

5. Use `get_logger(__name__)` from the local logging_config module for all logging.

**src/docproc/Dockerfile** ‚Äî Following the core-api Dockerfile pattern:

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# System dependencies for Docling layout models and EasyOCR
# libgl1 is NOT needed because we use opencv-python-headless (Pitfall 1)
# libglib2.0 is needed for some image processing operations
RUN apt-get update && apt-get install -y --no-install-recommends \
    libglib2.0-0 \
    && rm -rf /var/lib/apt/lists/*

# Install uv for fast dependency installation
RUN pip install --no-cache-dir uv

# Copy project metadata and install dependencies (cached layer)
COPY pyproject.toml .
RUN uv pip install --system --no-cache .

# Copy application code
COPY . .

# Create uploads directory (will be mounted as volume)
RUN mkdir -p /app/uploads

# Create EasyOCR model cache directory (persisted via volume)
RUN mkdir -p /root/.EasyOCR

EXPOSE 8001

HEALTHCHECK --interval=15s --timeout=5s --retries=3 --start-period=120s \
  CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8001/health')"

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8001"]
```

Key details:
- Port 8001 (not 8000, which is core-api)
- start_period=120s (longer than core-api's 30s because Docling + EasyOCR model loading is slow on first boot)
- No libGL dependency ‚Äî we use opencv-python-headless per Pitfall 1
- EasyOCR cache dir created for model persistence
- libglib2.0-0 is needed by opencv-python-headless for some operations
